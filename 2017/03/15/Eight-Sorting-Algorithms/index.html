八大排序算法

发表于 2017-03-15 | | 阅读次数

最近准备实习面试，花了两天时间把八大经典排序算法整理了一遍，按照 思想 - 步骤 - 动画演示 - Python 实现 - 性能分析 的顺序进行总结。

选择排序

简单选择排序

思想：

每次循环在队列中选择最小的元素，放到指定的位置。

步骤：

  1. 在未排序的序列中，选出最小的元素和序列的首位元素交换
  2. 在剩下的未排序序列中，再选出最小元素与序列的第二位元素交换
  3. 以此类推，直到所有元素排序完毕

堆排序

思想：

利用最大堆，输出栈顶元素，即最大值，将剩余元素重新生成最大堆，继续输出堆顶元素，重复此过程，直到全部元素都已输出。

步骤：

  1. 构造最大堆：数组下标范围是 0~n-1，因为叶结点的元素是最大堆，因此，从下标 n/2 开始的元素均为最大堆。因此，从 n/2-1 开始，依次向前构造最大堆，这样可以保证，当构造某个节点时，它的左右子树都已经是最大堆了。
  2. 堆排序：因为堆使用数组模拟的，因此得到一个最大堆后，数组内部并不是有序的。因此需要将数组有序化。思想是：每次都移除最大堆的根节点，然后在对剩下的元素进行最大堆调整。即，第一次将 heap[0] 与 heap[n-1] 元素交换，然后对 heap[0 … n-2] 做最大堆调整，第二次将 heap[0] 与 heap[n-2] 元素交换，再对 heap[0 … n-3] 做最大堆调整。重复该操作直至 heap[0] 与 heap[1] 交换。操作完毕后的整个数组就是有序的了。
  3. 最大堆调整：一个辅助函数，提供给上面两个方法调用。目的是将指定区间的元素进行最大堆调整，使得子节点永远小于父节点，这样最大值就会出现在根节点处。

插入排序

简单插入排序

思想：

将待排序的一组序列，分为 已排好序 和 未排序 两个部分。每次循环，将未排序序列中的元素与已排好序序列中元素比较，选择合适的位置插入。

步骤：

  1. 从第一个元素开始，该元素可以认为是已经被排序
  2. 取出下一个元素，在已经排序的元素序列中从后向前扫描
  3. 如果被扫描的元素（已排序）大于新元素，该元素后移一位
  4. 重复步骤3，直到找到已排序的元素小于或者等于新元素的位置
  5. 将新元素插入到该位置
  6. 重复步骤2~5，直到所有元素排序完毕

交换排序

冒泡排序

思想：

进行 N-1 次循环，在第 k 次循环中，从 1 到 N-k 个元素从前向后比较，如果顺序错误，就进行交换。这样每次循环就像是把最大的元素沉到了最后面，或将最小的元素冒到了最前面（冒泡）。

步骤：

  1. 比较相邻的元素。如果前一个元素比后一个大，那么就进行交换
  2. 对第0个和n-1个数据进行比较。这样，最大的数就沉到了数组的最后
  3. 保持已经沉到最低的位置上的数据不动，对剩下的元素重复步骤2
  4. 以此类推，直到所有元素都排序完毕

快速排序

思想：

快速采用了 分治法 (Divide-and-ConquerMethod) 的思想，将未排序元素根据一个作为基准的 “主元” (Pivot) 分为两个子序列，其中一个子序列的记录均大于主元，而另一个子序列均小于主元，然后递归地对这两个子序列用类似的方法进行排序。

步骤：

  1. 选择一个主元，并与最后一个元素交换
  2. 设置两个指针 Low 和 High，初始值分别指向第一和倒数第二个元素
  3. Low 从左向右扫描，若遇到比主元大的元素则停止；High 从右到左扫描，若遇到比主元小的元素则停止
  4. 如果 Low 和 High 没有错位，则两个元素交换
  5. 重复步骤3~4，直至 Low 和 High 错位，将基准与 Low 对应的数交换
  6. 递归地对以基准划分的左右两子序列用同样的方法进行快排

归并排序

思想：

归并排序也可以看作是分治法的应用。主要思想是：先递归分解数组，再合并数组。

步骤：

  1. 先考虑合并两个有序数组：比较两个数组的最前面的数，谁小就取谁，取了后相应的指针就往后移一位。然后再比较，直至有一个数组为空，最后将另外一个数组剩余的元素复制过来。
  2. 再考虑递归分解：将数组分成 left 和 right 两个数组，如果这两个数组内部是有序的，那么就可以步骤1的方法将其合并。如何保证这两个数组是有序的呢？可以再二分，直到分解出的数组只含有一个元素为止，因为认为这一个元素的数组内部已有序。然后再进行合并。

基数/桶排序

思想：

用若干个桶来帮助进行排序的排序算法。该算法不需要进行数值之间的大小比较，只需要将数字每次按照对应位放入相应的桶中。分为主位优先法（MSD）和次位优先法（LSD）。

步骤：

  1. 设置一个定量的数组当做空桶
  2. 从低位开始，将当前位数字相同的数字放入对应的桶中
  3. 将桶内所有数字按顺序重新按高一位进行排序
  4. 重复进行直至所有数字中最高位的数字排完

额外空间复杂度 O(N+R)

其中，N - 关键字个数，R - 桶的个数，D - 分配收集的趟数（按基数分解后的位数）。
